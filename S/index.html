公司簡介

翔威國際成立於1997年，位於台北市文山區羅斯福路六段218號，核心成員來自資策會，主要股東為華碩電腦集團及三商電腦。目前業務經營範圍包括資訊軟體之產品開發、銷售與專業服務。除台灣總公司外，美國及中國大陸均設有分支機構。且為了加強中國業務及前瞻性市場開發，現已在蘇州、東莞及深圳等地成立業務據點。

公司約100人左右，分為總經理室、總管理部、多語化服務事業部、MES事業部、資訊部、研發部、技術服務事業部等七個部門。

翔威國際致力於發展全方位軟體解決方案、開發大型企業客戶訂製類軟體，以及套裝軟體與IA嵌入式軟體。目前為擴展亞洲區軟體版圖，積極研發自有品牌應用軟體產品，並進行國外產品的本土化工程。

工作項目簡述

Intranet

以WordPress建立的公司內部網站，配合Javascript及API完成頁面細部功能。主要放置的內容為行事曆、公告、管理代號、員工座位、分機號碼、電子郵件…等對內公開資訊。

表單提醒工具

以WPF(Windows Presentation Foundation)架構製作的應用程式，能夠於用戶收到新表單時發出提示訊息。

MIS例行檢查表

以網頁填寫送出形式取代檔案上傳，配合API將記錄儲存至資料庫(MongoDB)中，也可檢視過去的紀錄。

RESTful API

REST指的是 "Representational State Transfer"，「具象狀態傳輸」。和一般API不同的是，RESTful API善用HTTP Verb來制定簡潔直觀的URI，並以Web可接受的資料類型(如：JSON)來作為物件進行操作。

eForm

eForm為數位表單系統，員工可使用此系統傳送請假單、訂購單、中英文證明申請單…等數十種表單。eForm也包含管理介面，匯集過去分散於各系統的功能，管理者可於eForm管理介面中新增/修改/刪除員工、部門、行事曆…等資訊。

工作詳述

  • 舊版Intranet

    內容管理系統(CMS, Content Management System)，為一種創建網站的工具，可將相關內容集中儲存並具有群組管理、版本控制等功能。

    使用者可在後臺發布頁面、文章，也可使用上傳檔案、設定帳戶等功能。常見的CMS有WordPress, Drupal及Joomla!。

    公司內部網站命名為「Intranet」。正如其名，只有使用公司內部IP才能夠連上此網站。先前的Intranet使用Drupal 7建置，使用LDAP(輕型目錄存取協定，Lightweight Directory Access Protocol，似小型資料庫)儲存資料，並配合PHP將資料顯示於網頁中。

    但擔心CMS升級之後會出現難以解決的問題，且不升級也能正常運作，故Intranet的CMS一直維持在安裝時的版本，效能及安全性都比較差。新版Intranet以Javascript和API取代舊版Intranet中的PHP各項功能，並以Bootstrap美化版面。

  • RESTful API

    為了從客戶端(網頁、應用程式…等)取得資料庫中的資料，我們必須撰寫API並於客戶端呼叫某些方法才能達成。我們都是使用Visual Studio做為開發工具，並以C#開發API。

    為了增加安全性，除了限制唯有內部IP才可呼叫之外，我們也在每套API都加上了OAuth2認證機制(JWT Token驗證)。客戶端存取資料時必須於header中提供認證碼，認證通過之後才可存取資料。

    而欲取得認證碼，客戶端必須於首次登入時提供其登入eForm時使用的帳號及密碼，待驗證成功後便會將回傳的token值存入LocalStorage (為HTML5中的資料暫存區)，之後便可於header中使用此認證碼。

    開發API時，我們也必須撰寫API規格書。這些API可能會在多個地方被使用，故需要將其詳細記錄下來。每次修改過後必須寫上更新紀錄，以便之後的開發者閱讀及修改。

  • MIS例行檢查表

    例行檢查包含每日和每月檢查，範圍廣泛，包含：攝影機狀況、網路狀況、網站狀況、網路電話狀況、虛擬機運作情形、網路流量…等超過20個項目。

    原先的紀錄方式是以Word書寫，完成之後匯成pdf格式上傳至檔案伺服器。但此過程過於繁瑣，且若這些數據能置於資料庫中，將來也可用以進行數據分析，故我們以網頁型式，搭配MongoDB重新製作例行檢查表，省下建立/上傳檔案的時間。

  • 表單提醒工具

    員工在eForm上填寫的表單(如：請假單、各式申請單…等)經送出後需由特定對象審核。但是公司當時並沒有能夠通知對象收到新表單的工具或系統，導致表單常常過了許久才被審核。

    因此我們以WPF(Windows Presentation Foundation)製作了表單提醒工具，並搭配先前寫的API，在用戶收到新表單時發出通知。

    表單提醒工具以ClickOnce發行，每當程式開啟時便會自動更新，我們只需將最新版本發行至伺服器上即可，如此使用者便不需自行下載。登入過後也會將登入資訊存在AppData中，防止資料被刪除，以利下次進行自動登入。

  • eForm電子表單

    eForm為翔威國際數位表單系統，第一代的電子表單於2000年獲行政院新聞局頒發「傑出資訊應用及產品獎」。採用表單核心的思考邏輯，首創系統架構制度化，以狀態改變技術運作機制，不僅能快速建立電子表單，加速系統導入時程，更能保障系統之穩定性。

    但舊版eForm唯有在IE上才可正常運作，且資料庫中也有許多欄位已不再使用，但先前的維護人員因擔心改動之後前端頁面的功能會發生異常，便決定不更動先前的欄位，導致資料庫中的資料重複性高，資料表也越來越複雜。

    此次開發的是第二代電子表單，前端使用Angular 4，後端則使用ASP.NET Core來實作。我們重新設計了許多資料表，整合各表單間共通的狀態碼並建立相關文件，詳細記錄各項資訊以利之後的開發者進行維護。

    我們也為表單建立了許多控制項，如：人員選擇元件(Userpicker)、日期選擇元件(Datepicker)…等，每一張表單都可以使用這些元件。並將表單共用函式整合至同一個class中，以繼承的方式來建立新表單。同時也建立製作表單的SOP，使製作新表單變得簡單、快速。

ABOUT Dimerco

中菲行國際物流集團自總公司（中菲行國際物流股份有限公司）1971年成立以來即專注於國際運輸及相關服務之經營，在貨運承攬市場上已奠定良好的競爭利基。集團更"致力於提供客戶服務、整合與顧問三合一的全球物流運籌服務，成為最具競爭力的國際物流運輸服務公司"。

中菲行放眼國際，掌握趨勢，全球行銷網路超過140個服務據點遍佈亞洲、北美洲、歐洲、澳洲等17個國家，並在其餘各國與代理商結盟，締交超過190個策略夥伴服務據點。透過策略聯盟, 更與航空公司、船公司建立夥伴關係，整合上中下游供應鏈，以提供客戶全方位物流運輸服務，協助客戶找到最佳的物流解決方案，控管成本且及時送達。

Be the most competitive transportation and logistics service provider, the key elements of Dimerco Express Group growth strategy are global reach with local touch in network expansion, products diversification and operation optimization, following data integration along with the application of Internet of Things (IoT) in building physical and virtual organization to provide far reaching International Logistics Services.

ABOUT Value Plus System

近年來集團以大中華經濟圈為母體市場的經營策略，積極擴展全球行銷服務網，業績持續成長；並建立全球即時資訊管理系統( Dimerco Value Plus System©-中菲行增值訊息系統) 及電子商務服務平臺，不但強化內部資訊整合提高集團經營管理績效，並透過多元化網站互動功能及整合性資料庫提供客戶供應鏈管理所需之即時資訊，以協助客戶在市場上的競爭能力。

中菲行整合性的電子商務服務平臺及全球及時資訊 ( Dimerco Value Plus System ©-中菲行增值訊息系統) 已完成版權的注冊；不僅能提供客戶創新的IT解決方案，並可及時傳遞及整合資訊，並依客戶的要求為客戶提供更有效率的管理。中菲行不斷的提昇及強化Dimerco Value Plus System © ，現階段此（1）個整合系統包含（2）個資料庫、（3）個資訊平臺、（7）個功能模組及（5）個延伸管理系統。透過此系統不僅提昇公司經營績效，更可以協助客戶的供應鏈管理，來降低其經營成本提昇效益。

ABOUT MVC

此次實習目的是將Dimerco Value Plus System 從2.0的ASP.NET Web form 模式轉換成效益於管理的 MVC 模式。MVC模式（Model–view–controller）是軟體工程中的一種軟體架構模式，把軟體系統分為三個基本部分：模型（Model）、視圖（View）和控制器（Controller）。 MVC模式最早由Trygve Reenskaug在1978年提出，是全錄帕羅奧多研究中心（Xerox PARC）在20世紀80年代為程式語言Smalltalk發明的一種軟體架構。

MVC 模式的應用程式的目的就是希望打破以往應用程式使用的大雜燴程式撰寫方式，並間接誘使開發人員以更高的架構導向思維來思考應用程式的設計，因此對於一個剛入門的初學者來說，架構導向的思考會有一定的門檻，需要較多的實作與練習才能具備相應的能力，大多數的初學者還是較習慣於大雜燴式的程式撰寫，所以可能會對 MVC 模式抱持著排斥或厭惡的心態，然而 MVC是有助於應用程式長遠的發展，雖然大雜燴式的程式也可以用來發展長生命週期的應用程式，但是相較於 MVC，大雜燴式的程式在可擴充性和可維護性 (尤其是可測試性) 上會遠比 MVC 複雜很多，相反的，MVC 模式的應用程式是在初始開發時期必須先思考並使用軟體架構，使得開發時期會需要花較多心力，但是一旦應用程式完成後，可擴充性、可維護性和可測試性反而會因為 MVC 的特性而變得容易

The purpose of the MVC model is to implement a dynamic programming that simplifies the subsequent modification and extension of the program and makes it possible to reuse a part of the program. In addition, this model makes the program structure more intuitive by simplifying the complexity.

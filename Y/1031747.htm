一、 工作內容

    我目前在IT部門下的ERP部門中的DC(Data Center)組，主要的工作是負責FAWeb內部企業網站的網頁開發與維護。日月光中壢(ASECL)的IT 資訊系統區分為兩大類 CIM 與 ERP。 CIM電腦整合製造包括 MES 製造執行系統與 FA工廠自動化。接收來自產線作業員過帳資料與機台連線的資料,以提供給後端系統做資料統計與分析。ERP是企業資源規劃系統，ASECL 在 2012年導入 SAP 五大模組 (MM/PP/SD/FI/CO) 以提供Supporting 人員更嚴謹也更完善的系統功能。2014年 ASECL 導入 Dispatch 派料系統以提升工廠效率。並於2015年著手 Planning 投料輔助系統.以提升客戶滿意度。以上的系統資料會根據內部管理人員與外部客戶需求,設定排程將資料彙整到 Data Center (HDB database)，整理好的資訊再透過 FAWeb內部企業網站提供給各級主管掌握工廠的運作情況。針對客戶的需求,依據客戶的IT系統規格書,整理成客戶需要的資訊到 ASED3 (B2B database) 以滿足客戶B2B的需求。

上圖為FAWeb的登入畫面
    FAWEB 的Report 是以組織做為分類大項。以產線來說主要有Assy(Assembly) / FT(Final Test) / CP(Wafer Probe) / WLCSP (Wafer Level CSP)。產線的管理報表又可依功能區分為以下幾項對應到 Report ID。
    a.    FCT是 CycleTime Report，FEQ 是 Equipment 設備相關報表。(Dispatch 雖是與機台相關,但獨立拉一個類別)
    b.    FWP(指的是 WIP working in process) 生產報表對應到 Hold / WIP 兩個類別。FOT 指的是Output 產量分析報表。
    c.    FYL 指的是 Yield Report (製程良率分析報表)。
    d.    FPP (指的是 Piece Part Report) 封裝物料報表。
    非產線直接相關的組織有 PC 生管 (Production Control)，QA 品管 (Quality Assurance)，CRM (Customer Relation Management)客戶關係管理，IT 部門 ，FA (IT 工廠自動化)，MC 物管 (Material Control) 等等…PDD(Product Definition Data) 報表，是提供User 查詢 MES 製造系統，產品生產的相關資訊。依據不同製程再分為 Production 正式區與 Buffer 暫存區。命名有 PAB，PAP PSP等等…
    SAP類別的報表，依據各不同模組 再區分為 SMM / SPP /SSD 三類Report。
網頁的 Coding 都依循 DC team 所提供的 asp.net c# coding standard開發。
HDB (History data base) 透過在DB 開發 ETL 程式將前端系統 (SAP/MES) 資料彙整到資料倉儲作為FAWeb網頁查詢的來源。
ETL 指的是 Extact/ Transfer/ Load。透過 Extract 篩選條件跨 DB-Link 取得資料，接著根據 User 所提供的商業邏輯(Business Rule) 將資料整理成有用的資訊。像是 WIP，Output，Cycletime，Yield。最後是 Load 將資訊存入資料表中。

    FAWeb除了正式區外，還有一個測試區名為FAWeb-QAS，QAS的環境與正式區基本上是一樣的，差別在於連到的DB不同，主要的用途是讓程式上線前先在QAS中經過測試，確認程式沒問題後才會上線到正式區，以免程式出錯不小心新增刪除修改到了不該變更的資料。

二、 學習

        從二月開始實習到現在三個半月了，覺得自己的程式設計能力進步很多，而且以前不太熟的SQL語法現在也越來越上手。也學到了很多公司內部運作的流程，以及公司系統的架構，像是程式的版本該如何控管、寫好的程式該如何上線…等。我相信這些都是在學校學不到的。除了專業技能外我還學到了如何跟同事相處，以及做人處事的方式。我變得比以前更敢發問，求知慾也比以前更強。除此之外我的責任心也變得更重，常常有許多報表都需要在期限內完成，而且主管交代的事情我都會如期完成。另外我的時間管理能力也變得比以前更好。

三、 自我評估及心得感想

        剛開始進來上班時很怕自己什麼都不會，怕主管交代的事情做不出來，還好主管有安排一系列的教學課程，讓我對公司更加瞭解，工作起來也更加順手。 開始上班後，才會發覺以前在學校裡學到的很多東西都只是皮毛而已。像是SQL的語法，在學校學的頂多就是SELECT、FROM、WHERE，大部分的查詢都可以在十行內解決。而接觸到公司的SQL後，動不動就是三、四百行，又有很多判斷條件將SQL串在一起，每次都看得眼花撩亂。另外也要特別感謝學長姐不厭其煩的指導我們，我們提出的問題他們通常都有問必答，若是遇到他們不會的也可以一起研究一起學習。公司裡的工程師也都很熱心，都會來協助我們解決問題，像是寫網頁最常遇到的就是自己寫出來的跟公司規範的coding standard不一致，這時候就需要請教工程師或學長姐如果按照公司的寫法要怎麼寫，若是不按照公司的coding standard，可能會造成效能低落，或是系統不易維護等問題。我覺得我還需要加強的SQL的部分，覺得自己還不夠熟悉，常常需要上網查才寫的出SQL。

二、學習心得

        進公司實習半年後，已經對網頁和資料庫有一定的熟悉程度了；做事的效率也比以前提升了非常多。但是在暑假主管開始讓我們學習寫ETL(將資料從來源端經過萃取（extract）、轉置（transform）、載入（load）至目標資料庫的過程)；這對我們來說是一個全新的東西。上了四堂兩小時的課後，還好也順利通過公司內部的測驗。現在已經能夠自己寫一支完整的ETL，看著資料成功匯入資料庫，非常有成就感。另外主管也將網頁壓力測試的工作交付給我，我和學長姐研究出做壓力測試的方法，也成功測出有些程式的效能很差。

三、 自我評估及心得感想

        除了前端的網頁報表，最近也開始接觸系統內部像是ETL、procedure等。從一開始寫的程式常常因為不符合公司的coding standard 而被工程師退件，到現在每支上線的報表都是由我來檢查是否符合規範。希望未來的半年裡我能夠加強自己寫ETL的能力，並且開始幫公司撰寫與維護ETL。等到下一屆學弟妹近來實習時，希望我有足夠的能力能夠把他們教好。

四、 對繫上的建議

        我覺得資管的學生普遍上來說程式與資料庫的能力跟資工系的比起來還是差一截。繫上應該可以多開一些進階的程式設計或資料庫相關課程，給有興趣讓自己能力更上一層樓的同學選修。很感謝繫上給我這個機會能進到日月光這樣的大公司實習，讓我能夠提早適應職場生活，未來進入職場能夠更沒有壓力。

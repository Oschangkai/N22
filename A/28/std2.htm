(2)工作內容

交接完後一兩個星期我都在練習java，稍稍熟悉了之後主管給了我幾萬筆資料，是外面廠商的一些文本資料要我和數位文化中心的資料庫進行比對。資料內容為一關鍵詞、及關鍵詞前後的十五個字，我的工作則是比對資料與資料庫中文本的異同之處。由於資料庫存放的是一段一段的文本我必須從文本中找出關鍵字，再擷取出關鍵詞前後之十五個字，接著才能進行比對。除了比對之外還要列出文本的書名、段落及關鍵字是否配對成功，看似很簡單的文本比對，其實需要透過許多繁複的步驟，且也有許多的例外。首先是讀取文本，由於不清楚外面廠商是如何產生這些資料，因此也就無法確定廠商是如何取前後十五字的，那些資料出現了缺少前後十五字、十五字中包含特殊符號、超過指定字數等例外狀況，各個狀況都要一一處理。再來是從資料庫擷取文本，由於資料庫存放的並非單純的書籍內文，還包含著專家和學者的註解及一些xml標籤，因此需要處理過才能作後續的動作。處理完許許多多的例外之後即可開始進行比對。透過資料庫查詢語法取得包含有關鍵詞的前50筆資料，再從各筆資料擷取關鍵詞的前後十五個字比對它與廠商給的資料的符合程度，此時又產生一個問題，在同個文本中有可能出現兩次以上的關鍵詞，也就代表我必須對同個文本進行多次以上的搜尋，才能比對出最正確的結果。剛開始我是以逐字比對的方式得到一個分數代表其符合程度，可是發現這種方法效果並不理想，後來經由主管建議研究最長共同子序列 (Longest Common Subsequence) 並使用主管給我的一個比對程式運用史密斯-沃特曼演算法(Smith-Waterman algorithm)來產生比對分數，此時的結果就趨近於我們所想要的結果。完成了文本比對之後，主管接著給了我下一份工作。

這份工作是文本分析，我們處理的文本皆屬於文言文，由於我們並不具備透過直接觀看文本來得知那些詞語或詞句是重要的，因此得透過程式分析。以自己設定的條件去產出一些結果並在之後交由專業學者去判斷其結果之效用。我們最終的目的是找出文本內重要的詞句或重要的結構，例如：法規條文、公文皆有他們的書寫結構。我目前在做的是嘗試找出重要詞句。我的處理流程為：取得文本後我們以標點符號斷句，以每一個句子當成一個關鍵詞句組成全文的斷句陣列，由於句子可能重複所以我將重複的句子去除產生關鍵字陣列，接著進行交互比對。我們認為長度三到八的詞語應當最具有意義，因此只產出關鍵字長度為三至八的結果。至於比對方式，我將以關鍵字為基準取出頭尾相同，中間可能有差異的詞語，並計算其出現頻率。例如： Keyword = 三十年 Result = 三千年 三百年 三十年。 而要找出差異，本來我是使用上述的LCS，可是並無法準確的取得長度正確的詞語，所以改用Google開發的google-diff-match-patch套件。完成了交互比對之後，要做的就是將結果進行篩選，因為當關鍵詞出現相似的詞例如：十二年、十七年等，結果就會被重複計算。而我衡量詞與重要性的方法是以熵(entropy)的方式。最後將所有結果輸出至資料庫以供查詢，包含關鍵詞、出現頻率、及與類似詞語合併計算的熵。完成了重點詞句的比對，之後便是處理長度較長的句子，處理流程一樣如上圖，有差異的部分是產生關鍵字陣列的部分。我並非是取整個句子為一關鍵詞，而是以特定長度每次橫移一個字產生所有這個句子可能產生的詞語例如：

學習

(1) Python

Python是一款物件導向的直譯式語言，包含完備的函式庫，能輕鬆的完成常見的工作。它的語法簡單，透過縮牌來定義區塊。Python的創始人為Guido van Rossum，它對Python的定義是「優雅」、「明確」、「簡單」，適合用於處理單件事物。在剛進入公司時，學長推薦我們可以嘗試學習Python，透過Codecademy網站的練習，很快的懂得如何撰寫Python程式語言。它寫起來真的如創始人所描述的簡單及優雅，由於要求縮排，因此Python的程式碼寫起來即是整整齊齊的，它拿掉了傳統的括號及分號，避免了許多因人為疏失而產生的意外。也不需要對變數進行宣告，正因為不需要宣告，對變數的處理就要更加小心，避免發生不預期的錯誤。資工系學長所研究的影像辨識就是使用Python的程式與openCV輔助套件。

(2) Java

Java是一款在學校未曾接觸的程式語言，它的風格十分接近C與C++，繼承了C++物件導向技術的核心，捨棄容易引發錯誤的指標功能。由James Gosling所設計。它具有跨平台、泛型程式設計的特性使他被廣泛印用於Web應用開發及App開發。它不同於一般語言，它會先將原始碼編譯成位元組碼，後在依各不同平台上的虛擬機器來解釋執行因此實現了「一次編寫，到處執行」的跨平台特性。它與傳統的程式語言不同，它兼容許多java開發商所撰寫的套件，使java功能越來越強大、越來越完善。昇陽電腦也就是James Gosling所任職的公司對Java語言的解釋是：「Java程式語言是個簡單、物件導向、分散式、解釋性、健壯、安全與系統無關、可移植、高效能、多執行緒和動態的語言」。 在實習期間，由於主管們大部分結是使用java進行開發，為求程式能共兼容，本來我想用Python來完成主管交予的工作，因此轉而研究起java。後經過一小段時間的練習，發現其實java是非常好上手的，撰寫容易、功能完善、還有許多套件能提升你的撰寫效率，加上許多的例外處理讓你很容易進行debug，也不需要小心翼翼的處理指標問題，我覺得java是款非常適合有點基礎的人學習的程式語言，且用途也很廣泛。

(4) Recurrent Neural Network

Recurent Neural Network(RNN)遞歸神經網路，為類神經網路的一種，為機器學習的一部份。要了解RNN首先就得了解機器學習是什麼，機器學習主要是設計和分析一些讓電腦可以自動「學習」的演算法，從資料中歸納或整理出一些規律或資訊，並透過規律對未知的資料進行預測，機器學習大致上可分為兩類：監督式學習及非監督式學習，監督式學習主要是從訓練資料中學習到一個模式(learning model)並依此來對新的資料進行判斷，而非監督式學習則相反，不需要人工標籤，而是對數據進行據類。 van Rossum，它對Python的定義是「優雅」、「明確」、「簡單」，適合用於處理單件事物。在剛進入公司時，學長推薦我們可以嘗試學習Python，透過Codecademy網站的練習，很快的懂得如何撰寫Python程式語言。它寫起來真的如創始人所描述的簡單及優雅，由於要求縮排，因此Python的程式碼寫起來即是整整齊齊的，它拿掉了傳統的括號及分號，避免了許多因人為疏失而產生的意外。也不需要對變數進行宣告，正因為不需要宣告，對變數的處理就要更加小心，避免發生不預期的錯誤。資工系學長所研究的影像辨識就是使用Python的程式與openCV輔助套件。

類神經網路又稱人工神經網路是一種運用節點相連來形成一個類似生物神經的結構，能夠學習歸納，也就是說能夠通過已知資料的實驗運用來進行學習，它能夠推理產生一個可以自動識別的系統，此外它們也具有推理功能，只是它們是建立在邏輯演算法的基礎上，也就是說它們之所以能夠推理，基礎是需要有一個推理演算法則的集合。其基本的架構有三個部分：輸入層（Input layer）、輸出層（Output layer）、及隱藏層（Hidden layer）隱藏層介於輸入及輸出之間，可以是一層也可以是多層。

而RNN則是類神經網路的其中一種，以下將介紹兩種RNN的形式，一種為最簡單的SimpleRNN，另一個為更進階的LSTM，由於LSTM的效果更好一些，因此現在講RNN通常都是只LSTM。 SimpleRNN的結構如下圖：

(simpleRNN)

圖的左邊為一個RNN細胞，其中x為輸入、o為輸出、s為存於細胞中的值U、V、W為一權重，控制值的計算，值得注意的是，W部分代表的是一個暫存記憶，裡面的值會成為下一個時間點的s。右邊則為不同時間點t-1、t、t+1單個RNN網路的運作方式，假設U的計算方式為乘算、W為加算、V為乘算，此時在t-1的時間點輸入為1，細胞中的值也是1，此時在計算輸出時便是1乘1而W的暫存值為1加1。下個時間點我們同樣輸入1，此時細胞中的值已被更新為2因此計算出的輸出即為2。 而學者將SimpleRNN改良發明出LSTM，範例如下：

其最大的特點在於多加了三個門檻值用來操控神經資訊的更新，最早的SimpleRNN由於細胞中的值一直進行更新，因此較舊的資料就無法對現時的輸入產生影響。 這種保有過去計算資料的神經模式很適合用於具有時間性或順序性的資料處理，RNN最出色的應用便是在手寫字的影像辨識上，也可以做語音辨識、翻譯等模型訓練。

(5) 如何撰寫論文

我覺得寫論文算是蠻困難的一件事，首先得閱讀許多與你所做的研究相關的論文，撰寫時在用語上也得很嚴謹，且最好能進行舉例，讓閱讀者能更好的讀懂論文，此外好的撰寫結構也很重要，內文要緊緊的扣住主題，這次試著撰寫論文讓我學習到許多論文的撰寫技巧。

(6) 如何做出好的報告

由於我們幾乎每周三都要進行一次進度報告，因此可以藉此磨練報告的技巧，我們平時很清楚自己的工作，可是如何講解給其他人聽就要考驗報告的功力。PPT如何做的讓人一目瞭然、講解如何讓人能聽得懂，這都是我必須磨練的。剛開始報告的不清不楚，被組長校正，他告訴了我們許多的技巧。 首先，若報告中出現圖片，則得明白的標示出圖片究竟代表什麼、或者圖片所呈現的是哪一個部分。然後在報告時，要在PPT中指出自己在講的位置，可以用Highlight、表框、箭頭等方式指出，並且若發現自己無法在一個頁面中完整的述說或解釋完整個概念，可以將它分成許多個，即使PPT頁面看起來很單調，只要能清楚的講完你所想表達的概念即可。再來是PPT標題的命名，標題代表了這張投影片所想表達的內容核心，即使是介紹同樣的東西，也不會讓標題重複，至少要在後面新增加(Continue)讓看的人能知道這是話題的延續。最後是回顧，由於組長及主管們平時都很忙，也有許多自己的事要處理，也不一定能記住我們上一次報告究竟做了哪些東西，因此最好要在報告現在的進度前先進行上次的回顧，好讓主管了解你目前究竟在做些什麼，並銜接到現在的進度。

自我評估及感想

首先是對於自己的評估，我覺得自己花了很大的心力在工作上面，並很努力的嘗試著完成工作。雖然我們的工作壓力並不是很大，也沒有需要做出產品、上班時間很彈性、主管也不怎麼管我們，可是相較於資工系我們在程式方面的功力實在是差距太大了，因此不認真可不行。從二月二號實習至今，我每天都準時的上下班，不遲到、不早退，這是對工作的基本要求。而當工作需要什麼技能，可是我並不會時，我就必須去自行學習，而當自己在工作上有問題或是對主管的要求有不清楚的部分，我就會去請教主管，以確保我做的事符合主管的要求。若組長因為公務忙碌而沒有進行進度會議時，我會找時間去向主管報告我前的進度，讓主管好掌握我究竟在做什麼。缺點方面，由於有時會要閱讀論文，或對程式進行debug而其過程通常枯燥乏味，在加上還有課堂上的報告或考試要顧，有時會出現稍稍恍神的情形，但不影響工作的進度。綜合上述所說，我認為自己在工作上的表現還算不錯。

心得感想方面，我覺得來中研院數位文化中心實習是很幸運的一件事，它就像學校的延伸，不只能學習到許多新知，相較於學校多了更多實作的過程。由於數位文化中心的定位是在為保存好的文化資料進行創新應用及保存再利用，在這裡工作讓我了解到究竟我們有多少文化資產並沒有正確的被使用，也缺乏大家來對它進行活化以提供大眾觀看或進行研究。同時要成功的將文化資產推銷給觀眾也是一門艱深的課題，大部分的民眾對於一些冷門知識並沒有那麼熱衷，要能成功讓這些文化資產成功進入大眾眼裡就得以平易近人的方式來實行像是漫畫或小說。又或者要提供文化資產供學者們研究，如何提供好的工具也是一個困難點，舊時代運用肉眼和紙張書寫的研究方式太沒有效率了，且也無法保證精準度。在這個資訊時代，唯有運用科技、電腦幫助學者進行研究才是正確的方式。而影像辨識、網頁標註、文字探勘都是其中的一環。

我很喜歡這裡工作的氣氛，大致上算是輕鬆愉快，自由度也很高。聽到其他朋友實習的狀況，常常需要跑來跑去忙東忙西的、又或者要產出一些產品，與我們相比之下壓力就較大。我覺得實習最主要的目的就是要學到有用的技能、了解市場目前的需求，讓自己在畢業之後與社會在銜接上有著較少的衝擊。來到中研院我也確實的學到許多新技能，也懂得去哪裡才能獲取更多的知識，順便也探索一下自己的興趣。主管及學長們也對我們很好，有問題而他們知道如何解決的話大多會回答我們，若是不知道的話就會大家一起討論，看有沒有辦法想出辦法解決。所以我覺得來數位文化中心工作室非常值得的。

暑假期間我大多在進行程式的撰寫，理解到專案的開發真的是一件很困難的事，平時我們撰寫小程式還好，大專案的分工及程式的合併就需要各成員的磨合，有時候小地方的改動便會對整個專案有很大的影響，或者必須請他人進行改動。這次撰寫的語言主要是JS及JSP，也是我較不熟悉的語言，因此有許多不足的地方，但大致還可以勝任。

隨著專案慢慢開發，為了增加更多的功能，很多時候需要修改後台的程式，這次碰到最大的困難是程式邏輯上的問題，由於要處理的文本狀況很多，應付的狀況也很多，常常很難以一種方法對文本進行全面的處理，因此在處理方法上花了許久的時間，不過近期也解決了，接下來便是要處理效能的問題，未來我將與主管繼續討論。

對系上的建議

感謝系上有這麼好的機會提供我們實習，幫我們尋找廠商還提供面試場地，並告訴我們面試的準備技巧。

而我想建議系上，希望在安排資管系課程的時候能加入多更多程式實作的部分，讓我們做更多的練習，提升我們的程式技巧。然後希望在實習期間可以減輕一點課程負擔，由於校外實習規定要三天在公司工作，不太有時間能處理課業的事物，再加上若是課程有許多的報告要做也很比較難空出時間進行團隊討論，造成組員困擾。最後希望這份實習報告可以稍微減少規定的頁數。
